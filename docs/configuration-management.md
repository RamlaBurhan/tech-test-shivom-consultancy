# Configuration management

Ansible configures the hosts that Terraform provisions and runs the application container.

## Roles

- **common**: updates the apt cache, installs base packages and creates the application and log directories.
- **docker**: adds the official Docker apt repository and GPG key, installs the engine, CLI, containerd and the compose and buildx plugins, then enables the service. The repository architecture is chosen from `ansible_architecture` so the same role works on amd64 and arm64.
- **app**: syncs the application source (or pulls a published image), renders a compose file from a template and brings the stack up with `community.docker.docker_compose_v2`, then waits on the health endpoint before finishing. A node-exporter container is included so the host is scrapeable by Prometheus.

The `app` role works in two modes set by `app_build_locally`. When true it builds from synced source, which is how the local VM test runs. When false it pulls `app_image`, which is how the CD pipeline deploys the image built in the pipeline.

## Inventories

- `inventory/local.ini` generated by the multipass script for the local VM.
- `inventory/aws_ec2.yml` dynamic inventory that discovers running instances tagged `Project=sample-web-app`.

## Testing

The multipass script provisions a real Ubuntu VM and runs the full playbook including the app role, then curls the health and info endpoints on the VM:

```bash
make vm
```

Linting is `yamllint` plus `ansible-lint` at the production profile, both run in CI on every push and pull request.

Run it locally before pushing:

```bash
make ansible-lint
```
